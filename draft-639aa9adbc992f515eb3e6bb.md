# What is event-driven architecture?

Event-driven architecture is a software paradigm in which decoupled systems, applications, and services interact through events and triggers actions in response.

Now there are many definitions out there for event-driven architecture, and all of them convey the following characteristics:

* Events are used as a communication mechanism. Events are published and subscribed asynchronously via an event broker.
    
* Systems are independent and hence can be developed and deployed separately.

## Benefits

There are many benefits to consider when building an event-driven architecture, and important to note is that the build cost outweighs these benefits.

One of the main benefits is the overall flexibility and the ability to scale at ease. As the systems are decoupled, new features can be added, and existing ones can be modified without disrupting the overall functionality. The same can be said for scaling the solution.
